What ILM is

Layer 1: per-script projection tables map every writing system to a canonical hub. Layer 2: a fixed reversible kernel (Romenagri, GPL 2003) emits ASCII-7, case-free, diacritic-free output that is legal as a C identifier — so native-script source compiles through an unmodified GCC, and the surface is always recoverable. One canonical identity, every layer of computing.

Per-language programming standards

68 languages × 201 constructs — real translations (Bengali যদি, Tamil எனில், Urdu اگر), each with its reversible Romenagri ASCII-7 identifier form, published as per-language standards documents that map into ISO/IEC 9899 (C), 14882 (C++), IEEE 1800, IEEE 754, Khronos, OMG, ONNX — localization of the standards, not re-authorship. Browse the standards · keyword packs · machine-readable status

Scripts vs languages — two axes, both covered

Scripts (writing systems): 45 Brahmi-derived projection tables + Perso-Arabic family + NW-Semitic (Hebrew table live; Syriac/Aramaic/Phoenician/Samaritan staged) — Layer-1, generated and flagged. Languages (vocabulary): the 68-language keyword registry — Layer-"speech", each language in its own words. A Bengali programmer gets Bengali words in Bengali script; the kernel beneath never changes.
