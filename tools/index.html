PyPIDOI

misty-doi

Automation-first DOI minting — one canonical metadata record fans out to Zenodo/DataCite/codemeta/CITATION.cff, with OpenTimestamps provenance. No backend, no lock-in.
pipx install misty-doi
misty init -o metadata.json
misty publish -m metadata.json -f paper.pdf      # --dry-run / --sandbox / production

spi-scan

Sensitive-Personal-Information scanner for a working tree, full git history, and GitHub metadata: emails, AWS/GCP keys, GitHub PATs, PyPI/Zenodo/Slack tokens, private keys, JWTs, phones, IPs, home paths, Aadhaar, cards, passports. Exit 7 on any HIGH finding. Run it before transferring or open-sourcing anything.
# installs from git (not yet on PyPI):
pipx install git+https://github.com/project-ilm/spi-scan.git
spi-scan path ./repo          # working tree
spi-scan git  ./repo          # full history, all branches
spi-scan gh   owner/repo      # issues, PRs, releases
spi-scan all  ./repo owner/repo

opspublishingpatent

Workflow engines

The actual publish and patent workflows, built on misty-doi. Each item is a tracked directory with a status ledger.
# in project-ilm/ops/bin
publish_paper.sh new my-paper        # init -> validate -> stamp -> doi -> submission -> track
patent_track.sh  new my-invention    # disclosure -> OTS proof -> prior-art -> filing package (counsel files)

opsconvergenceresilience

Convergence & resilience toolchain

Discover every org/repo, generate a convergence plan from real data, dedupe locally, gate for SPI before announcing, mirror across forges, and keep it all running.
# in project-ilm/ops/bin
discover_all.sh                      # enumerate all orgs/repos -> inventory + plan
dedupe_scan.sh ~/work                # duplicate files (report-only)
pre_announce_scan.sh --org project-ilm   # SPI go/no-go before announcing
mirror_forges.sh --org project-ilm   # GitLab/Codeberg/Savannah/OpenForge mirrors
convergence_driver.sh --watch 6      # run the whole sweep continuously
