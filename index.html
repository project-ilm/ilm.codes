OPERATING CONTRACTS
  - Self-contained artifacts: scripts WRITE their files (cat>fn / base64), never
    check for files an author forgot to ship. Binaries travel with them.
  - Declared environment: PEP 668 hosts install CLIs via pipx (venv alt), never
    bare pip into system Python.
  - Credentials by path, on the executing host only — never in artifacts/prompts.
    If you lack a token or cannot reach a service, say so and hand over the one
    host-side command; do not pretend to have published or pushed.
  - Side effects run where the credential lives. Safe by default (no flags = no
    side effects); irreversible steps explicit, confirmed, idempotent.
  - All GitHub writes via fork → PR → merge, LOUD output. Verify before mutating.

Confirm you read the five sources and restate the contracts before any change.

The hypothesis

Script ≠ Language ≠ Standard ≠ Representation. Beneath all of them there may be a single canonical computational identity Ω that survives projection — recoverable, reversible, and script-independent. If so, localizing the entire stack stops being an impossible per-language effort and becomes one fixed kernel plus additive tables.

N tables + 1 kernel

A script is one presentation table (Unicode → Devanagari hub → Romenagri ASCII-7). A language is one ontology row. The kernel never changes — never N².

HindiC++ — a language standard, not a hack

In hindawiai/chintamani: an ISO-style specification defining HindiC++ as a Hindi lexical form of C++ by token equivalence. मुख्य→main, पूर्णांक→int, द्विगुन→double. The meaning is the corresponding C++ translation unit — same syntax, same semantics, recoverable surface. This is the langspec process made formal: anyone can write a standard for their language, dialect, or idiolect.
